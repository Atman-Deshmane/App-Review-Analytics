## Weekly App Review Pulse: Groww Leadership Briefing
**Reporting Period:** 15 Jun 2026 to 28 Jun 2026

### Executive Summary
Feature requests continue to dominate user feedback, with a significant impact score driven by requests for specific technical indicators. While app performance concerns are surfacing with high user engagement, a notable negative sentiment is emerging around charges and fees, directly impacting user retention.

### 1. Top Themes & Strategic Insights

*   **Feature Requests (Impact Score: 5723, Review Count: 22):** This theme represents the overwhelming majority of user-driven impact. The primary driver is a highly specific request for the addition of the stochastic indicator, with a user actively chasing this feature for over a year. This indicates a strong demand from a segment of our user base for advanced trading tools.
*   **Other (Impact Score: 176, Review Count: 191):** While this theme has the highest review count, its impact score is significantly lower, suggesting a broad range of less critical issues. However, a particularly concerning negative quote points to severe customer support failures, which can severely damage brand reputation.
*   **User Interface (Impact Score: 141, Review Count: 46):** Recent UI updates are generating frustration. Users are experiencing lag, slow performance, and unexpected behavior after interaction, particularly with equity rows. This indicates a potential regression in user experience with the latest version.

### 2. Voice of the Customer (Quotes)

*   **"please add stochastic indicator, which is very useful, I am chasing from last one year still not adding why??"** (Votes: 5650) - This quote highlights the extreme demand and user frustration regarding missing key trading indicators, directly impacting feature development priority.
*   **"The app is user-friendly but the brokege is very high compared to the competition. brokege should come down to 10 Rs per lot. don't go for this app."** (Votes: 5) - This quote directly articulates a significant competitive disadvantage in our pricing structure, leading to explicit recommendations against using the app.
*   **"worst update in 17.91.1 version as it's UI is not like previous one.. it should be when we touch equity row with price row..ie first row anywhere it should be open.. but is taking only left hand side if we touch equity.. UI is very lagging and slow.. plz improve app speed.."** (Votes: 50) - This feedback points to critical usability issues and performance degradation in the latest UI update, directly affecting the day-to-day user experience.

### 3. Recommended Actions

*   **Prioritize Stochastic Indicator Development:** Given the overwhelming impact and user demand, immediately assess the feasibility and timeline for integrating the stochastic indicator into the app.
*   **Investigate and Address UI/Performance Regression:** Conduct an urgent review of the recent UI update (version 17.91.1) to identify and resolve the reported lag, slowness, and interaction issues.
*   **Competitive Pricing Analysis for Brokerage:** Initiate a thorough competitive analysis of brokerage charges, particularly for lot-based trading, and evaluate the potential for adjusting our fees to remain competitive and reduce churn.
